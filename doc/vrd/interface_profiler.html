Interface - Views

The Visual Remote Debugger interface consists of a number of windows, or views. These views all interpret the same data, the scene, but displays it in different ways. The different types of views are:

Event List
Play Controls
Profiler
Scene Browser
Viewport



Profiler
The Profiler provides a way to access and visualize timing info from the input stream. To the left, a list of timing entries are shown, grouped under the scene they belong to. A time entry typically looks like this:

PrSimulate - 11761 cycles in 1 calls (11761 cycles per call)

The color indicates the total amount of time taken for a certain task in relation to the other tasks. A fully blue entry takes 100% of the total time, and a white entry takes 0%.

Clicking an entry will display it in the profiler graph to the right. This graph can be used to see how performance varies over time to detect spikes, or just get an overview over how the load is distributed over time. The graph will automatically change its scale to fit the current graph.
