Interface - Views

The Visual Remote Debugger interface consists of a number of windows, or views. These views all interpret the same data, the scene, but displays it in different ways. The different types of views are:

Event List
Play Controls
Profiler
Scene Browser
Viewport



Event List
The event list is the most "hardcore" views of them all. It displays the raw events captured from the stream, with some help information for easier reading. Normally, this information is not very useful, but there are cases where the actual course of events may be interesting for debugging purposes. Especially when debugging the actual stream writing itself (e.g. when custom objects added to the debugging stream).

There are six event types available (see Developer reference for details about event usage), and each has its own format in the list. Typically object types are displayed in parentheses, IDs are prefixed by # and names are in quotes. For example, a Set Parameter event may look like this:

Set parameter "Velocity" (Vector) in object with ID #a081588 (NxActor).

The information displayed in the event list is based on the current state of the scene, which means that types and names for objects that have not been created yet, or have been removed, can not be resolved. In these cases, Unknown is displayed instead of the name or type.

Doubleclicking an event will cause the debugger to run up to, or roll back to that event. The background color of the events show whether they have been executed or not. A light gray background means the event has been executed, while a white background means the event has not been executed yet.

Navigation

As an assistance, when navigating in the event list, there are some functions available in the event panel:
  • Search
    Using the search box, you can enter any text that is displayed in the event list, and search from the current position, either up or down in the list. The search is case insensitive. Note that the text displayed for events depends on the current event stream position (e.g. objects can change name when playing forward).

  • Auto focus
    When selected, will automatically focus the event list on the current position in the event stream.

  • Frames
    Enter an integer, and the event list will focus on the n'th frame break event. Please note that this is not the same as the number of simulated frames when running a multi-scene application. If you are positioned within the event stream and want to find out the current frame number, either alt+click any item in the list, or double click a frame break event and the current frame number will be shown in the "Frames" text box.
