Interface - Views

The Visual Remote Debugger interface consists of a number of windows, or views. These views all interpret the same data, the scene, but displays it in different ways. The different types of views are:

Event List
Play Controls
Profiler
Scene Browser
Viewport



Scene Browser
The scene browser can be used to get detailed information about any object in the scene. The scene appears as a tree with expandable branches where each node is an object in the scene or a parameter thereof. Due to performance issues, the browser will not update itself when the debugger is currently playing back a stream, although it will update while singlestepping.


The objects in the tree appear on the following format:

NameTypeID
[<name> - ]NxActor - #c6353b0

The name is optional, and only displayed when a Name string parameter is found on the object. Parameters are shown in blue, and appear like this:

NameValue
Angular velocity:  (0.02, 1.59, -3.42)

Parameters can be one of a number of different types (see Developer reference for details), all of which are displayed in a straight forward manner (strings as strings of text and numbers as numbers). There are a number of special cases though. In the example above, there is a Vector parameter, displayed as three scalars separated by commas and enclosed by parentheses. Matrix parameters are displayed similarily, although in this case there are nine scalars separated into three lines:

Mass local orientation:  ( 0.97, -0.23,  0.00
  0.23,  0.97,  0.00
  0.00,  0.00,  1.00)

Frame parameters are combinations of the two, with the origin of the frame as a Vector and the orientation as a Matrix:

Frame:  (-0.60,  0.00,  0.00)
( 1.00,  0.00,  0.00
  0.00,  1.00,  0.00
  0.00,  0.00,  1.00)

Finally, Binary data parameters only display their amount of data. A Binary parameter therefore looks like this:

Triangle data: 580b binary data



By clicking an object in the browser, this object will be selected, centered and tracked in the viewport, provided it has a resolvable graphical representation. Right clicking an object will bring up a context menu:

Hide <object> in viewport
Hide all other objects
Show only intersecting objects
Show only intersecting objects (enlarged bounds)
Show all objects in viewport
Watch <object>
Watch <object> in new browser
Return to root
Export Mesh

Hide <object>
This option will hide the selected object and all its children from the viewport. If the object is already hidden, the menu item will read "Show ..." instead and can be used to reveal the object again.

Hide all other objects
This option will hide all the objects except the selected one.

Show only intersecting objects
If this option is selected, all objects will be hidden except the selected one and others which intersected with it.

Show only intersecting objects (enlarged bounds)
This option is almost same as [Show only intersecting objects], but the bounds used to count the intersecting is enlarged.

Show all objects in viewport
If this option is selected, all objects in viewport will be unhidden.

Watch <object>
This option will reroot the browser window to the selected object

Watch <object> in new browser
This option will open a new browser window, rooted at the selected object.

Return to root
This option is only available if the browser currently already is rooted at another object than the real root, and will bring the browser back to the root.

Export mesh
This option is only available to all mesh shapes, triangle meshes, convex meshes and heightfields. When this option is selected, a [save as] dialog will pop up, and you can save the mesh as an Wavefront .obj file.
