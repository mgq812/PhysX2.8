Interface - Views

The Visual Remote Debugger interface consists of a number of windows, or views. These views all interpret the same data, the scene, but displays it in different ways. The different types of views are:

Event List
Play Controls
Profiler
Scene Browser
Viewport



Viewport
The viewport is the most high level view of the scene. Objects are displayed graphically to give an intuitive overview of the state, which makes it easy to grasp spatial relationships.

There are two primary ways to navigate the scene; Orbit mode and Fly mode. These modes can be selected in the View > Camera menu. In Orbit mode, the camera orbits around a certain point in space. The following key/mouse combinations can be used to move the camera:

ALT + Drag Left Mouse ButtonOrbit camera around viewpoint
ALT + Drag Middle Mouse ButtonPan camera
ALT + Drag Right Mouse ButtonZoom camera in/out

In Fly mode, the view is controlled much like a fly mode in a first person shooter game. The following key/mouse combinations can be used to move the camera in Fly mode:

WMove camera forwards
SMove camera backwards
AMove camera left
DMove camera right
Drag Right Mouse ButtonRotate camera


From the View > Camera menu, it is also possible to select a camera that has been detected in the data stream (see Developer reference for details). This will lock the view to the selected camera, and manual view movement is disabled.

Clicking an object with the left mouse button will select it, and the main Scene Browser will focus on that object. By double clicking an object when in Orbit mode, the view will track that object, meaning the camera will move so that the object is always in the center of the view. To release tracking, select another object or pan the camera. If the debugger is currently playing at the end of the stream, objects in the scene can be affected by clicking and dragging them with the left mouse button. If you do not want to affect objects when clicking them , you can hold down the ctrl key and the objects will only be selected.

Apart from drawing the actual geometries of the scene, the viewport can also visualize information about frames, velocities, joints, etc. Whether the different symbols should be drawn or not can be controlled from the View menu. In Selected mode, the symbols will only be drawn on the selected objects. The visual size of the symbols can be controlled from the Preferences dialog.
