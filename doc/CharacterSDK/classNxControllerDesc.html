NxUserControllerHitReport* NxControllerDesc::callback
 

Specifies a user callback interface.

This callback interface is called when the character collides with shapes and other characters.

Setting this to NULL disables callbacks.

Default: NULL

See also:
NxUserControllerHitReport

NxCCTInteractionFlag NxControllerDesc::interactionFlag
 

The interaction flag controls if a character controller collides with other controllers.

The default is to collide controllers.

Default: NXIF_INTERACTION_INCLUDE

NxF32 NxControllerDesc::skinWidth
 

The skin width used by the controller.

A "skin" around the controller is necessary to avoid numerical precision issues.

This is dependant on the scale of the users world, but should be a small, positive non zero value.

Default: 0.1

NxF32 NxControllerDesc::slopeLimit
 

The maximum slope which the character can walk up.

In general it is desirable to limit where the character can walk, in particular it is unrealistic for the character to be able to climb arbitary slopes.

The limit is expressed as the cosine of desired limit angle. A value of 0 disables this feature.

Default: 0.707

See also:
upDirection

NxF32 NxControllerDesc::stepOffset
 

Defines the maximum height of an obstacle which the character can climb.

A small value will mean that the character gets stuck and cannot walk up stairs etc, a value which is too large will mean that the character can climb over unrealistically high obstacles.

Default: 0.5

See also:
upDirection

NxHeightFieldAxis NxControllerDesc::upDirection
 

Specifies the 'up' direction.

In order to provide stepping functionality the SDK must be informed about the up direction.

  • NX_X => (1, 0, 0)
  • NX_Y => (0, 1, 0)
  • NX_Z => (0, 0, 1)
Default: NX_Y
