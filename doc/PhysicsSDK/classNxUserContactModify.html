Detailed Description

An interface class that the user can implement in order to modify contact constraints.

Threading: It is necessary to make this class thread safe as it will be called in the context of the simulation thread. It might also be necessary to make it reentrant, since some calls can be made by multi-threaded parts of the physics engine.

You can enable the use of this contact modification callback in two ways: 1. Raise the flag NX_AF_CONTACT_MODIFICATION on a per-actor basis. or 2. Set the flag NX_NOTIFY_CONTACT_MODIFICATION on a per-actor-pair basis.

Please note: + It is possible to raise the contact modification flags at any time. But the calls will not wake the actors up. + It is not possible to turn off the performance degradation by simply removing the callback from the scene, all flags need to be removed as well. + The contacts will only be reported as long as the actors are awake. There will be no callbacks while the actors are sleeping.

Member Enumeration Documentation

enum NxUserContactModify::NxContactConstraintChange
 

This enum is used for marking changes made to contact constraints in the NxUserContactModify callback. OR the values together when making multiple changes on the same contact.

Enumerator:
NX_CCC_NONE  No changes made.
NX_CCC_MINIMPULSE  Min impulse value changed.
NX_CCC_MAXIMPULSE  Max impulse value changed.
NX_CCC_ERROR  Error vector changed.
NX_CCC_TARGET  Target vector changed.
NX_CCC_LOCALPOSITION0  Local attachment position in shape 0 changed.
NX_CCC_LOCALPOSITION1  Local attachment position in shape 1 changed.
NX_CCC_LOCALORIENTATION0  Local orientation (normal, friction direction) in shape 0 changed.
NX_CCC_LOCALORIENTATION1  Local orientation (normal, friction direction) in shape 1 changed.
NX_CCC_STATICFRICTION0  Static friction parameter 0 changed. (Note: 0 does not have anything to do with shape 0/1).
NX_CCC_STATICFRICTION1  Static friction parameter 1 changed. (Note: 1 does not have anything to do with shape 0/1).
NX_CCC_DYNAMICFRICTION0  Dynamic friction parameter 0 changed. (Note: 0 does not have anything to do with shape 0/1).
NX_CCC_DYNAMICFRICTION1  Dynamic friction parameter 1 changed. (Note: 1 does not have anything to do with shape 0/1).
NX_CCC_RESTITUTION  Restitution value changed.
NX_CCC_FORCE32  Not a valid flag value, used by the enum to force the size to 32 bits.

virtual bool NxUserContactModify::onContactConstraint (  NxU32 &  changeFlags,
const NxShape *  shape0,
const NxShape *  shape1,
const NxU32  featureIndex0,
const NxU32  featureIndex1,
NxContactCallbackData &  data
)  [pure virtual]
 

This is called when a contact constraint is generated. Modify the parameters in order to affect the generated contact constraint. This callback needs to be both thread safe and reentrant.

Parameters:
changeFlags when making changes to the contact point, you must mark in this flag what changes have been made, see NxContactConstraintChange.
shape0 one of the two shapes in contact
shape1 the other shape
featureIndex0 feature on the first shape, which is in contact with the other shape
featureIndex1 feature on the second shape, which is in contact with the other shape
data contact constraint properties, for the user to change. Changes in this also requires changes in the changeFlags parameter.
Returns:
true if the contact point should be kept, false if it should be discarded.
Platform:
  • PC SW: Yes
  • PPU : Yes
  • PS3 : Yes
  • XB360: Yes
